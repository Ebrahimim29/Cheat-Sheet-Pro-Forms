238-معرفی فرمتهای مختلف جهت سریالازکردن دیتاهای فرم و معرفی آبجکت FormData:


روشهای ارسال اطلاعات به برنامه نویس بک اند (فرمتهای استاندارد ارسال اطلاعات به سمت سرور):


1-JSON: خوانا،ساختارمند و برای API های مدرن انتخاب اول محسوب میشه application/json => {"username":"abbas" , "password":"1234"}
2-UrlEncoded:ساده و سبک هست، برای فرمهای معمولی خیلی استفاده میشه application/x-www-form-urlencoded => username=abbas&password=1234
3-form-data: مناسب برای ارسال فایلها و داده های متنی با هم .multipart/form-data
4-text/plain: مناسب برای ارسال فایلها و داده های متنی با هم name=abbass
email=abbas@gmail.count-message

Serialize Data : تبدیل داده های فرم یا آبجکت ها به یک قالب استاندارد متنی که بشه اون رو به سرور فرستاد
